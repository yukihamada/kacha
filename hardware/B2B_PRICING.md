## 2. 少額短期保険会社とのバンドル設計

### 2-1. スキーム概要

```
[入居者]─────[KAGI デバイス]─────[不動産会社]
    │                │                │
  ¥200/月          データ          報告レポート
  保険料             │
    └──[少額短期保険会社]
           │
       孤独死特約 (発見費用保険金)
       見舞金: 最大 ¥1,000,000
```

KAGI 設置物件の入居者には孤独死特約付き少額短期保険を提供。
KAGI の生存確認データを保険会社と API 連携し、**保険料を最大 20% 割引** する。

### 2-2. 保険料割引の根拠計算

**孤独死発見遅延と損害額の相関 (業界データ)**

| 発見までの日数 | 特殊清掃費用 | 原状回復費用 | 合計損害額 | 発生率 |
|--------------|------------|------------|----------|--------|
| 1〜3日 (KAGI 検知) | ¥50,000 | ¥100,000 | ¥150,000 | 70% |
| 4〜7日 | ¥200,000 | ¥500,000 | ¥700,000 | 20% |
| 8〜30日 | ¥800,000 | ¥1,500,000 | ¥2,300,000 | 8% |
| 31日超 | ¥2,000,000 | ¥3,000,000 | ¥5,000,000 | 2% |

**KAGI 設置の期待損害低減効果**

```
設置なし期待損害額 = 0.70 × 150,000 + 0.20 × 700,000 + 0.08 × 2,300,000 + 0.02 × 5,000,000
                  = 105,000 + 140,000 + 184,000 + 100,000
                  = ¥529,000

KAGI 設置後 (24時間以内検知率 92% 想定):
設置後期待損害額 = 0.92 × 150,000 + 0.07 × 700,000 + 0.009 × 2,300,000 + 0.001 × 5,000,000
               = 138,000 + 49,000 + 20,700 + 5,000
               = ¥212,700

損害低減率 = (529,000 - 212,700) / 529,000 = 59.8%
保険料割引率 = 損害低減率 × 保険会社分配率 (0.33) ≒ 20%割引  ← 根拠
```

### 2-3. 保険会社への提案価値

- **引受保険料の低リスク化**: 孤独死特約の大規模リスクを KAGI がヘッジ
- **データ提供**: 月次の生存確認ログ (匿名化)、発見遅延ゼロ化実績
- **保険料収入の増加**: 割引で新規加入率 UP → 加入者数増加がリスク分散に寄与
- **目標提携先**: 少額短期保険 (孤独死担保): 東京海上日動あんしん、SBI日本少額短期保険、ハウスガード少額短期保険

## 3. UR都市機構・自治体向け調達スキーム

### 3-1. UR都市機構

**UR の課題**: 高齢単身入居者が全入居者の 35.7% (2023年時点)。孤独死対策は重要な経営課題。

**調達スキーム (推奨)**:

```
UR都市機構 (発注)
    │
    ├── 直接調達: 1,000台〜 / ¥6,500/台 + ¥480/月
    │
    └── 管理組合経由調達:
        各団地管理組合が独自に発注 → KAGI が一括管理画面を提供
        管理費から支出 → 住民負担ゼロ
```

**提案ポイント**:
- UR は `高齢者終身建物賃貸借` 制度の運営実績あり → KAGI とシナジー
- 厚生労働省 `孤独・孤立対策推進法` (2024年4月施行) への対応実績として PR 可能
- 調達要件: ISO 27001 認証相当のセキュリティ (ATECC608A 署名が証明)

### 3-2. 自治体向け (市区町村)

**補助金活用スキーム**:

|---------|--------|-------------|
| 地域生活支援事業 (厚労省) | 1/2〜2/3 | 高齢者見守りツールとして申請可 |

## 5. 介護報酬 ICT 加算 (2024年改定) との連携可能性

### 5-1. 関連する加算制度

**テクノロジー活用加算 (居宅介護支援)**
- 概要: ICT を活用した質の高いケアマネジメントへの加算
- 加算額: +50単位/月 (= 約 ¥575/月)
- 要件: 利用者情報の電子管理、事業所間のデータ連携

**科学的介護推進体制加算 (LIFE)**
- 概要: CHASE/LIFE への情報提供と PDCAサイクル実施
- 加算額: +40〜80単位/月
- KAGI との連携: 生活リズムデータを LIFE 項目に紐付け

**見守り機器等導入支援加算 (施設系)**
- 概要: 見守りセンサー等の ICT 機器を活用した夜間巡視の効率化
- 対象: 特別養護老人ホーム、グループホーム等
- KAGI の位置付け: 補助的見守り機器 (ATECC608A 署名付きログが証跡に)

### 5-2. KAGI × 介護事業者の提案

**対象サービス**: 訪問介護、居宅介護支援、グループホーム

```
[訪問介護事業者] ← KAGI API 連携 → [ケアマネ]
        │                                  │
    訪問頻度の最適化                  ICT 加算申請の根拠データ
    (KAGI が毎日の生存確認)           (生活リズムレポート)
```

**ユースケース**:
1. KAGI のセンサーログをケアマネの訪問計画に反映
2. 異常検知時は担当ヘルパーへ自動通知 → 安否確認訪問を追加
3. 月次の生活リズムレポートを LIFE フォーマットで出力

**価格 (介護事業者向け)**:
- デバイス: ¥8,500/台 (月払いオプション: ¥350/月 × 36ヶ月)
- クラウド: ¥480/月/台 + 介護連携オプション ¥200/月/台
- LIFE 連携 API: ¥30,000/月 (事業者単位、台数無制限)

## 6. 導入フロー

### 6-1. 標準導入フロー (不動産管理会社向け)

```
Week 1: 発注
  ├── 発注書 → KAGI 事務局
  ├── 請求書払い / クレジットカード / 銀行振込
  └── デバイス出荷 (最短 3営業日)

Week 2: 設置
  ├── 管理会社スタッフが各室に設置 (10分/室)
  │   設置場所: リビング/ダイニングの天井または壁 (マグネット固定)
  ├── QR コードで WiFi 設定 (入居者スマホ不要、管理者のみで完結)
  └── 設置完了報告 → KAGI ダッシュボードに反映

Week 3: 家族登録
  ├── 入居者 / 家族が KAGI アプリ (iOS/Android) をダウンロード
  ├── デバイス QR コードをスキャンしてペアリング
  ├── 通知設定 (24時間無応答でプッシュ通知 → 48時間で電話)
  └── 動作確認: I'm OK ボタンを押してアプリに通知が届くか確認

Week 4〜: モニタリング開始
  ├── 日次: 生存確認スコア (ACS) を KAGI ダッシュボードで一覧確認
  ├── 週次: 自動生成レポートをメールで受信
  ├── 異常時: アプリ / SMS / 電話の 3段階アラート
  └── 月次: 管理会社向け請求書 + 稼働レポート
```

### 6-2. 自治体・UR 向け大規模導入フロー

```
Phase 1 (Month 1-3): 実証実験
  ├── 対象: 50〜100台 / 1団地
  ├── KPI: 異常検知率、誤報率、入居者満足度
  └── 費用: 無償 or ¥1/月 (補助金申請の実績作り)

Phase 2 (Month 4-6): 評価・調達準備
  ├── 実証結果レポート作成 (KAGI が支援)
  ├── 入札仕様書のレビュー支援
  └── 補助金申請書への根拠データ提供

Phase 3 (Month 7-12): 本格調達
  ├── 入札 or 随意契約 (実証実績あり)
  ├── 一括納品 + 設置工事 (KAGI パートナー工事業者)
  └── 運用保守契約: SLA 99.5% / 月次レポート / 専任 CS
```
