| 指標 | Go条件 | No-Go → ピボット |

| Tier2誤報率 | < 0.5回/月/台 | > 2回: センサー構成変更 |

| リピート注文 | 10台以上 (追加物件) | 0: 製品がオーナーの課題を解いていない |

## リスクと対策

| リスク | 影響度 | 発生確率 | 対策 |
|--------|--------|---------|------|
| 誤報率が下がらない | 致命的 | 中 | フェーズ1で早期検証。ダメなら設計根本変更 |
| 大手参入 (Google/Amazon) | 高 | 低 | OSS戦略で差別化。アルゴリズムの先行優位 |
| 個人情報漏洩事故 | 致命的 | 低 | ATECC608A署名。音声/映像を記録しない設計。ISO 27001 |
| 自治体予算の凍結 | 高 | 中 | EC直販チャネルを並行して維持 |
| ESP32-S3の供給問題 | 中 | 低 | ESP32-C6への移行準備。マルチソース確保 |
| 法規制の変更 | 中 | 低 | 薬機法対象外を維持。福祉用具としての位置づけ |
| 利用者のプライバシー懸念 | 高 | 中 | レーダーのみ。カメラ/マイク録音なし。OSS公開で透明性 |

## 成功の定義

```
2026年末: KAGIが「動くこと」を証明する
  → 100台が民泊で稼働し、オーナーが「これは必要だ」と言っている

```

**「KAGIのおかげで助かった」という人が1人でもいれば、このプロジェクトは成功。**
**100,000台が稼働すれば、統計的にその人は確実に存在する。**
