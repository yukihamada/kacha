# KAGI Sensor Hub - 回路設計ノート (KiCad設計者向け)

## 1. ESP32-S3-MINI-1-N8 ピンアサイン

### I2Cバス (Bus 0) — 8デバイス
| 信号 | GPIO | 接続先 |
|------|------|--------|
| SDA | GPIO8 | SHT40, VEML7700, SGP41, LIS2DH12, BMP280, ATECC608A, SCD41, ST25DV |
| SCL | GPIO9 | SHT40, VEML7700, SGP41, LIS2DH12, BMP280, ATECC608A, SCD41, ST25DV |

全I2Cデバイスは同一バスに接続。アドレス競合なし (8デバイス全て固有):
| デバイス | I2Cアドレス | 備考 |
|---------|------------|------|
| SHT40 | 0x44 | 固定 |
| VEML7700 | 0x10 | 固定 (⚠️ BH1750の0x23とは異なる — ファームウェア要注意) |
| SGP41 | 0x59 | 固定 (SGP40と同一 — ドロップイン換装可) |
| LIS2DH12 | 0x19 | SDO/SA0 = HIGH |
| BMP280 | 0x76 | SDO = LOW |
| ATECC608A | 0x60 | 固定 (セキュアエレメント) |
| SCD41 | 0x62 | 固定 (CO₂センサー) |
| ST25DV04K | 0x53 | 固定 (NFCタグ I2C側) |

**プルアップ抵抗**: SDA/SCLそれぞれに **4.7K (R1, R2)** を3.3Vへプルアップ。
※10KΩでは400kHz時の立ち上がり時間が~600nsとなり規格(300ns)を超える。
 4.7KΩに変更することで~280nsに改善。100kHz(Standard)動作なら10KΩも可。
I2Cバスの総容量が400pF以下であることを確認。配線長は短く保つ。

### I2Sバス (マイク)
| 信号 | GPIO | 接続先 |
|------|------|--------|
| I2S_WS (LRCLK) | GPIO5 | INMP441 WS |
| I2S_SCK (BCLK) | GPIO6 | INMP441 SCK |
| I2S_SD (DATA) | GPIO7 | INMP441 SD |

INMP441の L/R ピン:
- GND接続 → 左チャンネル (WS=LOW時にデータ出力)
- VDD接続 → 右チャンネル
- **推奨**: GND接続 (左チャンネル)

## 2. 電源回路

### 2.1 USB-C → 3.3V メイン電源

```
USB-C (J1)
  VBUS (5V) ──┬── USBLC6-2SC6 (U10) ESD保護
              │
              ├── 100K (R7) ──┬── GPIO1 (ADC, 停電検知)
              │               │
              │              100K (R8) ── GND
              │               (分圧比 1:1 → 2.5V @ 5V入力)
              │
              ├── 22uF (C4) ── GND (入力バイパス)
              │
              ├── LD2410B VCC (5V直接給電, 10uF C6 + 100nF C2i)
              │
              └── ME6206A33M3G (U9)
                    VIN ── 5V
                    VOUT ── FB1 (600Ω@100MHz フェライトビーズ) ── 3.3V Rail
                                ├── 22uF (C3) ── GND (出力バイパス、FB1前)
                                │
                                └── 3.3V Rail (全IC・センサーへ)
```

**FB1 フェライトビーズ** (BLM18PG601SN1D, 0402):
- LDO出力とデジタル電源ラインの間に挿入
- 目的: mmWaveレーダー(LD2410B)とWiFi送信の高周波ノイズがセンサー電源ラインに乗るのを防止
- 特にSGP40(VOC)とSHT40(温湿度)への影響を低減
- DC抵抗: 0.4Ω以下 → 電圧降下<10mV @ 20mA (無視できるレベル)

### 2.2 CR2032 バックアップ回路

```
CR2032 (BT1, 3.0V)
  + ── 1N5819W (D2, カソード側が3.3Vレール) ──┬── 3.3V Rail
  - ── GND                                   │
                                           470μF (C5, 6.3V) ── GND
                                           ← WiFi TX瞬間電流バッファ

ME6206A出力 (3.3V) ── FB1 ──────────────────────── 3.3V Rail
```

**動作原理**:
- USB給電時: AMS1117が3.3Vを出力。D2は逆バイアスで遮断。CR2032は消費されない
- USB断時: AMS1117出力が低下。CR2032からD2を通じて3.3Vレールに給電
  - 1N5819Wの順方向電圧降下: 約0.3V → CR2032の3.0V - 0.3V = 2.7V供給
  - ESP32-S3の最低動作電圧: 2.3V → 十分なマージン
  - C5(470μF): WiFi TX時の300mAピーク電流を瞬時吸収。CR2032単体では内部抵抗(10Ω)により電圧崩壊するため必須
  - CR2032 + C5の組み合わせで約3〜5分の動作が可能 (BLE通知複数回送信可)

**⚠️ 停電時の通知戦略 (ファームウェア側で実装)**:
1. VBUS電圧監視(GPIO1 ADC)で停電検知
2. WiFiではなく**BLEアドバタイズ**で緊急通知 (15mA以下, CR2032対応)
3. BLE通知後はDeep Sleepに移行してCR2032を温存

### 2.3 USB-C コネクタ配線

```
USB-C (16pin, TYPE-C-31-M-12)
  VBUS (A4, B4) ── 結合 ── 5V Rail
  GND (A1, A12, B1, B12) ── 結合 ── GND
  CC1 (A5) ── 5.1K (R5) ── GND (UFPとして認識)
  CC2 (B5) ── 5.1K (R6) ── GND
  D+ (A6) ── USB_D+ (GPIO20)
  D- (A7) ── USB_D- (GPIO19)

  USBLC6-2SC6 接続:
    Pin 1 (I/O1) ── D-
    Pin 2 (GND) ── GND
    Pin 3 (I/O2) ── D+
    Pin 4 (I/O2) ── D+
    Pin 5 (VBUS) ── VBUS
    Pin 6 (I/O1) ── D-
```

**CC抵抗 5.1K**: USB-C規格でUFP (Upstream Facing Port) として認識させるために必須。
これがないとホスト機器から電力供給されない。

## 3. センサー個別回路

### 3.1 SHT40 (温湿度)

```
        VDD ── 3.3V
        GND ── GND
        SDA ── GPIO8 (I2C SDA)
        SCL ── GPIO9 (I2C SCL)

デカップリング: 100nF (C2a) VDD-GND間、ICの直近に配置
I2Cアドレス: 0x44 (固定)
```

注意: SHT40は自己発熱を避けるため、連続測定ではなく間欠測定 (1Hz以下) を推奨。
基板上の発熱源 (ESP32, LDO) からなるべく離して配置する。
精度: 温度 ±0.2°C / 湿度 ±1.8%RH (民泊用途で十分)。

### 3.2 LD2410B (60GHz mmWave レーダー) ※PIRから変更

```
電源 (5V専用 ── VBUSから直接給電):
        VCC ── USB VBUS (5V) ──┬── 10uF (C6) ── GND  (バルクバイパス)
                               └── 100nF (C2i) ── GND  (デカップリング)
        GND ── GND

シンプル存在検知 (OUTピン):
        OUT ── GPIO4
        ※ LD2410B内部プルダウン内蔵、外部抵抗不要 (R3削除)
        ※ 在室=HIGH (3.3V), 不在=LOW

UART 詳細データ (UART1):
        TX ── GPIO15 (ESP32-S3 UART1_RX)  ← レーダーからデータ
        RX ── GPIO14 (ESP32-S3 UART1_TX)  → レーダーへコマンド
        ※ LD2410Bの論理レベルは3.3V: レベル変換不要
        ※ ボーレート: 256000 bps (デフォルト)

OUTピン動作:
        High持続時間: 検知継続中
        Low復帰: 最後の検知から約5秒後 (ファームウェアで調整可能)
```

**UARTコマンド例** (詳細データ取得):
```
// 存在・距離・信号強度を取得 (10秒ごとに実行)
// レスポンス: 動体距離(cm), 動体強度, 静止距離(cm), 静止強度
tx: 0xFD 0xFC 0xFB 0xFA 0x02 0x00 0xFF 0x00 0x01 0x00 0x04 0x03 0x02 0x01
```

**配置**: 基板中央付近でも問題なし (フレネルレンズ不要)。
ただし筐体前面のレーダー窓を**プラスチック素材のみ**にすること (金属は電波を遮断)。
検知エリア: 約60°のコーン形状、最大6m。推奨設置位置: 部屋中央付近の壁面。

**R3 (旧PIRプルダウン)**: LD2410BはOUT内部プルダウン内蔵のため**削除**。
コスト削減: -$0.002。基板スペース解放。

### 3.3 MC-38 (ドアセンサー)

```
        ケーブル線1 ── GPIO3
        ケーブル線2 ── GND

GPIO3に10Kプルアップ (R4) → 3.3V
磁石が近い: スイッチ閉 → GPIO3 = LOW
磁石が遠い: スイッチ開 → GPIO3 = HIGH (プルアップで)
```

JST PH 2ピンコネクタ (J2) で本体と接続。ケーブル長: 約50cm。

### 3.4 VEML7700 (照度)

```
        VCC ── 3.3V
        GND ── GND
        SDA ── GPIO8
        SCL ── GPIO9
        ADDR ── NC (アドレス 0x10 固定。ADDRピンは存在しない)
        INT  ── NC (割り込み: 未使用)

デカップリング: 100nF (C2b) VCC-GND間
I2Cアドレス: 0x10 (固定) ⚠️ BH1750の0x23とは異なる — ファームウェア要確認
測定範囲: 0.0036〜167,772 lux (ALS_GAIN + ALS_IT レジスタで自動調整)
通常設定: ALS_GAIN=1/8 + ALS_IT=25ms → 屋内照度センシングに最適
```

**ファームウェア初期化例** (Rust/esp-idf):
```rust
// VEML7700 設定レジスタ (ALS_CONF: addr 0x00)
// ALS_IT=25ms, ALS_GAIN=1/8, ALS_SD=0 (電源ON)
i2c.write(0x10, &[0x00, 0x00, 0x00])?;
// 読み取り: ALS データレジスタ (addr 0x04)
let raw = i2c.read_word(0x10, 0x04)?;
let lux = raw as f32 * 0.0288; // ALS_IT=25ms, GAIN=1/8 の解像度係数
```

### 3.5 SGP41 (VOC + NOx)

```
        VDD ── 3.3V
        GND ── GND
        SDA ── GPIO8
        SCL ── GPIO9

デカップリング: 100nF (C2c) VDD-GND間 + 10uF (C1a) VDD-GND間

湿度補正: SHT40の湿度・温度データをI2Cコマンドで渡してVOC/NOx Index精度を向上
NOx Index: SGP41固有の機能 (SGP40では不可)。1-500スケール、100=基準値。
NOx応用: 調理時のガスコンロ漏れ検知、交通量の多い物件での空気質評価
```

**配置**: 基板端の通気スリット付近に配置。周囲に発熱源を置かない。
初回電源投入後、約1時間のバーンイン (慣らし運転) が必要。

### 3.6 INMP441 (MEMSマイク)

```
        VDD ── 3.3V
        GND ── GND
        WS  ── GPIO5 (I2S Word Select / LRCLK)
        SCK ── GPIO6 (I2S Serial Clock / BCLK)
        SD  ── GPIO7 (I2S Serial Data)
        L/R ── GND (左チャンネル選択)

デカップリング: 100nF (C2d) VDD-GND間
```

**プライバシー設計**: ファームウェアでdB値 (RMS) のみ計算し、PCMデータは即破棄。
ハードウェアレベルでの録音防止は不可能だが、ファームウェアで対応。
筐体に「音声は記録されません」の表示を推奨。

### 3.7 LIS2DH12 (加速度)

```
        VDD_I/O ── 3.3V
        VDD ── 3.3V
        GND (複数ピン) ── GND
        SDA/SDI ── GPIO8 (I2C SDA)
        SCL/SPC ── GPIO9 (I2C SCL)
        SDO/SA0 ── 3.3V (アドレス 0x19)
        CS ── 3.3V (I2Cモード選択: CS=HIGHでI2C)
        INT1 ── GPIO13 (振動・動き検知割り込み ← ファームウェアで閾値設定)
        INT2 ── NC

デカップリング: 100nF (C2e) VDD-GND間, 100nF (C2f) VDD_I/O-GND間
```

**INT1 割り込みの活用** (ファームウェア側で設定):
- `CTRL_REG3`: INT1_AOI1=1 → Activity/Inactivity検知
- `CTRL_REG5`: LIR_INT1=1 → 割り込みラッチ (GPIOが保持される)
- `INT1_THS`: 閾値設定 (例: 0x10 = ~250mg → 盗難/取り外し検知)
- **メリット**: ESP32がDeep Sleepでも、振動でGPIO13 HIGHにより即座に起床可能
- **注意**: LIS2DH12 INT1ピンはデフォルトpush-pull出力 → 外部プルダウン不要

### 3.8 ATECC608A (セキュアエレメント) ← v1.0で標準搭載

```
        SDA ── GPIO8 (I2C SDA)
        SCL ── GPIO9 (I2C SCL)
        VCC ── 3.3V
        GND ── GND
        WAKE ── 3.3V (常時アクティブモード, 省電力不要の場合)
                または GPIO10 (Sleep/Wake制御、消費電流 <150nA sleep時)

デカップリング: 100nF (C2j) VCC-GND間 (IC直近、マスト)
I2Cアドレス: 0x60 (固定)
```

**なぜ必須か**:
- ESP32のFlashにMQTT秘密鍵を平文保存するのはセキュリティリスク
- ATECC608Aは鍵を**絶対に外部に出さない**ハードウェア保護領域で保管
- 鍵を使った署名・検証はATECC608A内部で完結 → 鍵漏洩が物理的に不可能

**KAGI Hubでの用途**:
1. **デバイス認証**: ECC P-256鍵ペアによる相互TLS認証 (MQTT over mTLS)
2. **OTA署名検証**: ファームウェア更新の真正性をECDSA署名で確認
3. **デバイスクローン防止**: 量産時に各デバイス固有IDをプロビジョニング
4. **センサーデータ署名**: データが正規デバイスから来たことをクラウドで検証

**製造時プロビジョニング**:
- JLCPCBでSMT後、ファームウェア書き込み時にATECC608A初期化スクリプト実行
- Microchip Trust&GO / Trust FLEX対応 → クラウド登録まで自動化可能

### 3.9 BMP280 (気圧)

```
        VDD ── 3.3V
        VDDIO ── 3.3V
        GND ── GND
        SDI ── GPIO8 (I2C SDA)
        SCK ── GPIO9 (I2C SCL)
        SDO ── GND (アドレス 0x76)
        CSB ── 3.3V (I2Cモード選択)

デカップリング: 100nF (C2g) VDD-GND間
I2Cアドレス: 0x76 (SDO=LOW)
```

### 3.10 SCD41 (CO₂センサー) ← 世界初の民泊CO₂ハブ

```
        VDD ── 3.3V
        GND ── GND
        SDA ── GPIO8
        SCL ── GPIO9
        SEL ── GND (I2Cモード選択。VCC=SPIモード)
        nINT ── NC (割り込み: 未使用)

デカップリング: 100nF (C2m) VDD直近 + 100μF (C7) VDD-GND間 (ヒーター電流バッファ必須)
I2Cアドレス: 0x62 (固定)
電源: 3.3V (2.4-5.5V対応)
測定サイクル: 最小5秒 (single-shot low-power mode推奨)
```

**⚠️ SCD41の重要設計注意事項**:
1. **熱隔離**: SCD41はレーザー加熱素子を内蔵。SGP41と同様に通気スリット付近に配置
2. **温度補正**: SCD41の自己発熱補正にSHT40の外気温を使用 (`set_temperature_offset()`)
3. **Automatic Self-Calibration (ASC)**: 週1回以上、新鮮な外気 (400ppm) にさらすことで自動校正
4. **初回使用**: 最初の20秒は電源安定待ち → 測定開始

**CO₂アラート閾値設計** (ファームウェア側):
| CO₂濃度 | 状態 | アクション |
|---------|------|----------|
| < 800ppm | 良好 | — |
| 800-1200ppm | 普通 | VOC併用でモニタリング継続 |
| 1200-2000ppm | 要換気 | ゲストに「窓を開けてください」通知 |
| > 2000ppm | 危険 | 即座にオーナー通知 + ゲストアラーム (BZ1) |

**なぜNest/Aqara/Ecobeeが全社未搭載か**:
- SCD41は$14と高価 (主要競合は$5以下のセンサーのみ搭載)
- KAGIはプレミアム価格帯でこれを正当化できる唯一のプロダクト

### 3.11 ST25DV04K (NFCダイナミックタグ)

```
        VCC ── 3.3V
        GND ── GND
        SDA ── GPIO8
        SCL ── GPIO9
        GPO ── GPIO21 (スマホがタッチした瞬間にLOW → 割り込み)
               47K プルアップ (R11) → 3.3V
        RF_IN ── PCB スパイラルアンテナコイル (3ターン, 約35mm径)
        RF_OUT ── PCB スパイラルアンテナコイル
        C_tune: 47pF (C8) RF_IN〜RF_OUT間 (アンテナ共振周波数整合)

I2Cアドレス: 0x53 (固定, セキュリティレジスタ側)
```

**PCBアンテナ設計**:
```
基板外周に沿って3ターンのスパイラルコイルを銅箔で描画:
  - 線幅: 0.3mm
  - 線間: 0.3mm
  - 内径: 20mm, 外径: 35mm
  - レイヤー: Top銅箔
  - キープアウト: アンテナ内部に他の銅箔を置かない
  ※ JLCPCB 4層基板なら Inner 1/2はアンテナ下も問題なし
```

**NFCユースケース**:
```
ゲスト到着:
  スマホをKAGI Hubにタッチ
    → ST25DV が "KAGI-Guest-XXXXX / Password: XXXXXXXX" をNDEFで配信
    → スマホがWiFiに自動接続
    → KAGIアプリに「ゲスト到着」イベント記録

ファームウェアは毎チェックイン時にパスワードを生成・ST25DV I2Cで書き換え
チェックアウト後は無効なパスワードに更新 → セキュリティ維持
```

### 3.12 水漏れセンサー回路

```
        J3ピン1 ── GPIO16
        J3ピン2 ── GND

GPIO16に10Kプルアップ (R10) → 3.3V
D4 (PRTR5V0U2X) でESD保護 (ドアセンサーD3と同一回路)

水がない: センサー開放 → GPIO16 = HIGH (プルアップで)
水あり:   センサー短絡 → GPIO16 = LOW
```

**外部センサー (LEAK1)**: 床置き抵抗式水漏れ検知マット
- 銅電極2本が水で短絡 → LOW信号
- 設置場所: トイレ下、洗面台下、エアコン室内機の真下
- ケーブル長: 1m (追加延長ケーブル別売り)
- 検知後: BZ1ブザー + プッシュ通知 (即座に)

---

### 3.13 ピエゾブザー (BZ1) と音響イベント検知

```
        BZ1 VCC ── 3.3V
        BZ1 GND ── GND
        BZ1 IN  ── GPIO17 (HIGH=鳴動)

デカップリング: 100nF (C2n) VCC-GND間
```

**BZ1 音声パターン** (ファームウェア):
| イベント | パターン | 意味 |
|---------|---------|------|
| CO₂アラート | 3回ビープ (100ms間隔) | 換気が必要 |
| 水漏れ検知 | 5回ビープ (50ms間隔) | 水漏れ! |
| 緊急アラート | 断続ビープ10回 | 安否確認アラーム |
| 起動確認 | 1回ビープ (500ms) | システム正常起動 |
| ペアリング成功 | 上昇音2回 | WiFi接続完了 |

**TinyML音響イベント検知** (INMP441で入力、BZ1は出力):
| 検知イベント | アルゴリズム | アクション |
|------------|------------|----------|
| 煙感知器のサイレン音 (3kHz断続) | FFTパターンマッチング | 即座にオーナー通知 + BZ1同期 |
| ガラス破損 (高周波インパルス) | RMSエンベロープ解析 | セキュリティアラート |
| 激しい口論 (声の断続) | 音量+パターン | パーティーアラート前兆 |
| 転倒音 (短時間衝撃+静寂) | LIS2DH12 INT1と組み合わせ | 安否確認トリガー |

## 4. LED インジケータ (WS2812B-2020 × 2)

```
D1 (フロント面):
        VDD ── 3.3V
        GND ── GND
        DIN ── GPIO38 ── R9(33Ω) ── D1 DIN
        DOUT ── D1b DIN (デイジーチェーン)

D1b (リア面):
        VDD ── 3.3V
        GND ── GND
        DIN ── D1 DOUT (チェーン接続)
        DOUT ── NC

デカップリング: 100nF (C2h) 各LED VDD-GND間
2個のLEDを同じGPIO38で制御 → ファームウェアは2個分のデータを送信
同色で統一する場合: pixel[0] = pixel[1] = score_color
異なる演出: pixel[0]=フロントアンビエント, pixel[1]=リアステータス
```

### LEDパターン (アンビエントライト兼ステータス)

**通常動作: 家のスコアを色で表現 (ゆっくり息するように点灯)**
| スコア | 色 | 輝度 | 演出 |
|--------|-----|------|------|
| 90-100点 | 深緑 (#00C896) | 低 (20%) | 呼吸 0.5Hz |
| 75-89点 | 青緑 (#00B4D8) | 低 (20%) | 呼吸 0.5Hz |
| 60-74点 | 琥珀 (#FFB700) | 中 (30%) | 呼吸 0.8Hz |
| 60点未満 | 珊瑚 (#FF6B6B) | 中 (40%) | 呼吸 1Hz |

**システムイベント:**
| パターン | 色 | 意味 |
|---------|-----|------|
| 高速点滅 (5Hz) | 白 | WiFi接続中 |
| 高速点滅 (5Hz) | 青 | ペアリングモード |
| 3回フラッシュ | 赤 | アラート発生 |
| 消灯 | - | Deep Sleepモード (CR2032動作中) |
| レインボー | 虹 | ファームウェア更新中 |

**意図**: 棚の上に置いたとき「あ、今日の空気いいな」と家族が自然に気づく
 → 数値を見なくてもスコアが伝わるアンビエント体験

## 5. ボタン

### リセットボタン (SW1) → ESP32-S3 ENピン
```
        EN ── 10K プルアップ ── 3.3V
        EN ── SW1 ── GND
        EN ── 100nF ── GND (チャタリング防止)
```
注意: GPIO0はESP32-S3のSTRAPPINGピン。起動時にLOWだとダウンロードモードに入る。
リセットボタンはENピンに接続し、GPIO0はファームウェア書き込み時のみ使用する。

### ペアリングボタン (SW2) → GPIO2
```
        GPIO2 ── SW2 ── GND
        (ESP32-S3の内部プルアップを有効化)
```
3秒長押し: WiFi APモード起動 (再プロビジョニング)
5秒長押し: 工場出荷時リセット
