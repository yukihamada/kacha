## 1. コンセプト

日本では毎年約**6.8万人**が誰にも気づかれないまま自宅で亡くなる (2024年警察庁統計)。
発見まで平均**17日**。これは技術的に解決できる問題だ。

KAGIは「孤独死ゼロ」を目指す社会インフラとして設計されている。
民泊オーナー向けハブとして市場に入り、独居老人・障害者・一人暮らし全般へ展開する。

### 設計判断の基準
> 「これは、一人で死にかけている人を助けるか？」
> この問いがYESなら実装する。NOなら上位モデルのオプションにとどめる。

### KAGI Hub の機能概要 (本ドキュメントのスコープ)

**10種のセンサー** (温湿度/CO₂/照度/VOC+NOx/マイク/加速度/気圧/mmWave/ドア/水漏れ) + BLEゲートウェイ + WiFiルーター機能を1台に集約。

```
家のスコア: 87点  ●
─────────────────────────────
呼吸・在室   検知中  ─ 静止状態 (18:30〜)
空気質      92点   ↑ CO₂ 680ppm 良好
温度快適度   85点   ─ 24.2°C
異常なし     ─      最終確認 3分前
```

**設計思想**: 電源を挿すだけで動く。設定はNFCタップ or QRコードでWiFi接続。
工事不要、賃貸物件・介護施設・民泊すべてに設置可能。

## 2. センサー一覧と詳細ユースケース

### 2.1 温湿度センサー (SHT40 - Sensirion)
- **精度**: 温度 ±0.2°C, 湿度 ±1.8% RH (民泊用途で十分)
- **インターフェース**: I2C (0x44)

**ユースケース**:
| シナリオ | トリガー条件 | アクション |
|---------|------------|----------|
| エアコン消し忘れ | チェックアウト後に室温が外気温と5°C以上乖離 | オーナーに通知 + SwitchBot連携でエアコンOFF |
| カビ防止 | 湿度が70%を2時間以上継続 | 換気推奨通知 + 除湿器ON指示 |
| ゲスト快適度 | 室温が18°C未満 or 30°C超 | ゲストに空調操作案内をプッシュ |
| 凍結防止 | 冬季に室温5°C以下 | 水道凍結防止ヒーターON + オーナー緊急通知 |
| エネルギーレポート | 日次の温度推移 | 月次レポートでエネルギー効率スコア算出 |

### 2.2 人感・存在検知センサー - 60GHz mmWave レーダー (LD2410B - Hi-Link)
- **方式**: 60GHz FMCW ミリ波レーダー (PIRから変更)
- **検知距離**: 存在検知 最大6m / 動体検知 最大6m
- **検知角度**: 水平60°
- **出力**: UART (設定・詳細データ) + OUT ピン (シンプル存在/非存在)
- **動作電圧**: 5V (USB VBUSから直接給電)
- **消費電流**: 約70mA (5V)
- **価格**: $2.00 (EKMC1603111 $4.80比 **-$2.80**)

**PIRとの決定的な違い**:
| 能力 | 旧PIR | **LD2410B mmWave** |
|------|-------|-------------------|
| 動いている人の検知 | ✅ | ✅ |
| **静止している人の検知** | ❌ | ✅ |
| **睡眠中・呼吸のみの検知** | ❌ | ✅ |
| ペット・カーテンの誤検知 | 多い | 少ない |
| 設置場所の制約 | フレネルレンズ窓必要 | 筐体内に完全収納可 |
| パッケージ | TO-5 (手はんだ必須) | SMD (自動実装可) |

**ユースケース**:
| シナリオ | トリガー条件 | アクション |
|---------|------------|----------|
| チェックアウト確認 | チェックアウト時刻後に30分間無人 | 「退室完了」としてステータス更新 |
| 不審者検知 | 空室期間中に存在検知 | オーナーに即座にプッシュ通知 + カメラ連動 |
| 自動照明 | 存在検知 + 照度が低い | SwitchBot/Hue連携でウェルカムライトON |
| **安否確認** | **ゲストが6時間以上動かない (睡眠/倒れた)** | **緊急連絡先へ安否確認通知** |
| **深夜の静止検知** | **深夜2時以降に存在あり・無反応2時間** | **救急要請の判断材料として通知** |
| 清掃完了確認 | 清掃スタッフ退出後に無人確認 | 「清掃完了・入室準備OK」に自動更新 |
| 定員超過推定 | 検知エリア内の反応強度分析 | 定員超過の可能性をアラート |

**ファームウェア実装**:
- OUTピン: 存在/非存在のシンプル検知 (GPIO4で割り込み処理)
- UART: 10秒ごとに詳細データ取得 (静止距離・動体距離・信号強度)
- 検知感度: UARTコマンドで動的調整可能 (部屋の大きさに応じて設定)

### 2.3 ドア開閉センサー (MC-38 リードスイッチ)
- **検知距離**: 磁石間 15-25mm
- **出力**: デジタル (GPIO)
- **取付**: 両面テープでドア枠に設置、本体とはケーブル接続

**ユースケース**:
| シナリオ | トリガー条件 | アクション |
|---------|------------|----------|
| 施錠バックアップ | ドア開閉なし + Sesameロック状態 | 施錠済み確認のダブルチェック |
| チェックイン検知 | チェックイン予定時刻前後のドア開閉 | 「ゲスト到着」通知 |
| 深夜の出入り | 22時-6時のドア開閉回数が異常 | オーナーにアラート |
| 施錠忘れ | ドア閉→10分経過しても未施錠 | ゲストにリマインダー通知 |
| 長期不在検知 | 連泊中に24時間ドア開閉なし | 安否確認の連絡提案 |

### 2.4 照度センサー (VEML7700 - Vishay)
- **測定範囲**: 0.0036〜167,772 lux (オートゲイン、BH1750より高ダイナミックレンジ)
- **インターフェース**: I2C (0x10) ⚠️ BH1750の0x23とは異なる

**ユースケース**:
| シナリオ | トリガー条件 | アクション |
|---------|------------|----------|
| 照明消し忘れ | チェックアウト後に50lux以上 | SwitchBot連携で全照明OFF |
| 自動照明シーン | 日没検知（照度低下） + 人感検知 | ウェルカムライトをON |
| 遮光カーテン確認 | チェックアウト後にカーテンが開いたまま | 清掃スタッフへの指示に追加 |
| 電気代最適化 | 日中の高照度時に照明ON | 「自然光で十分」の省エネ提案 |

### 2.5 CO₂センサー (SCD41 - Sensirion) ← 競合全社未搭載

- **測定範囲**: 400-5000ppm ±40ppm (真のCO₂、VOC間接推定ではない)
- **インターフェース**: I2C (0x62)
- **自己校正**: 毎週1回の外気参照で自動校正 (ユーザー操作不要)

**ユースケース**:
| シナリオ | トリガー条件 | アクション |
|---------|------------|----------|
| 換気アラート | CO₂ > 1200ppm が15分継続 | ゲストに「窓を5分開けてください」 |
| 定員超過検知 | CO₂上昇速度が急峻 (>50ppm/分) | 「定員超過の可能性」をオーナーに通知 |
| 深夜の安否確認補助 | mmWave+ドア閉+CO₂= ゲスト在室確認 | 安否確認の精度向上 (CO₂あり=生存確認) |
| 空気清浄機連動 | CO₂+VOC同時上昇 | SwitchBot空気清浄機を自動ON |
| 清掃完了確認 | CO₂が外気レベル(400ppm)に戻った | 「換気完了・次のゲスト受け入れ可」通知 |

**競合との差別化**: Nest/Ecobee/Aqara/SwitchBotハブ — 全て CO₂センサー非搭載。
KAGIのみが「本当の換気アラート」を実現できる。

### 2.5b VOC + NOxセンサー (SGP41 - Sensirion)
- **測定対象**: VOC Index (0-500)
- **インターフェース**: I2C (0x59)
- **SHT45と組み合わせて湿度補正が可能**

**ユースケース**:
| シナリオ | トリガー条件 | アクション |
|---------|------------|----------|
| 換気アラート | VOC Index > 200 を30分以上 | ゲストに「窓を開けてください」通知 |
| 喫煙検知 | VOC急上昇 + 煙パターン | 禁煙物件での違反アラート |
| 空気質スコア | 日次の空気質推移 | ゲスト向けに「空気質スコア」表示 |
| 料理検知 | VOC + 温度上昇パターン | 火の元確認リマインダー |
| アレルギー対策 | チェックイン前の空気質 | 空気清浄機の自動起動 |

### 2.6 騒音センサー - MEMS マイク (INMP441)
- **測定**: SPL (Sound Pressure Level) のみ。音声データは記録・送信しない
- **インターフェース**: I2S (GPIO)
- **プライバシー**: ファームウェアレベルでdB値のみ計算。PCMデータはバッファから即破棄

**ユースケース**:
| シナリオ | トリガー条件 | アクション |
|---------|------------|----------|
| パーティー防止 | 22時以降に80dB超が10分以上 | ゲストに警告通知 → オーナーにエスカレーション |
| 騒音レポート | 時間帯別の騒音レベル推移 | 近隣トラブル防止のエビデンス |
| 生活リズム推定 | 静寂→活動の遷移パターン | チェックアウト時刻の予測 |
| 設備異常 | 異常な低周波振動音の検知 | 設備故障の早期発見 |

### 2.7 加速度/振動センサー (LIS2DH12 - ST)
- **測定範囲**: ±2g/±4g/±8g/±16g
- **インターフェース**: I2C (0x19) / SPI
- **消費電流**: 2μA (低電力モード)

**ユースケース**:
| シナリオ | トリガー条件 | アクション |
|---------|------------|----------|
| 洗濯機終了検知 | 振動パターンの停止 | ゲストに「洗濯完了」通知 |
| 地震検知 | 加速度の急激な変化 | 物件の安全確認アラート |
| デバイス取り外し検知 | 設置後の姿勢変化 | 盗難/いたずら防止アラート |
| 建物振動モニタリング | 長期的な振動パターン | 構造的な問題の早期発見 |

### 2.8 気圧センサー (BMP280 - Bosch)
- **測定範囲**: 300-1100 hPa
- **精度**: ±1 hPa
- **インターフェース**: I2C (0x76)

**ユースケース**:
| シナリオ | トリガー条件 | アクション |
|---------|------------|----------|
| 窓開放検知 | 急激な気圧変化 | エアコン稼働中なら「窓を閉めてください」通知 |
| 天気予報連動 | 気圧低下トレンド | 「雨が近づいています。洗濯物注意」通知 |
| 高度推定 | 設置フロアの推定 | マルチフロア物件での位置管理 |

## 2.9 WiFiルーター/ゲートウェイ機能 (AP+STA モード)

ESP32-S3のAP+STA同時モードを活用した **スマートWiFiゲートウェイ** 機能:

```
インターネット
     ↓
[自宅ルーター / ISPモデム]
     ↓ WiFi (STA接続)
[KAGI Hub]
     ├── WiFi AP → "KAGI-Guest-XXXXX" (ゲスト専用ネット)
     ├── WiFi AP → "KAGI-IoT" (IoTデバイス隔離ネット)
     └── BLE Gateway → SwitchBot / Sesame / Hue
```

**ユースケース**:
| シナリオ | トリガー | アクション |
|---------|---------|-----------|
| チェックイン自動検知 | ゲストスマホがKAGI-Guestに接続 | 「ゲスト到着」確定通知 (mmWaveとダブル確認) |
| ゲストWiFi自動制御 | チェックアウト時刻経過 | KAGI-Guest SSIDを自動停止 |
| 退室確認精度向上 | WiFiデバイス切断 + mmWave反応なし | 「確実な退室」判定 |
| IoTネットワーク分離 | 常時 | SwitchBot等をゲストネットから隔離 |
| 帯域異常検知 | 深夜の高トラフィック | 不審な大量通信をオーナーに通知 |

**追加ハードウェアなし (v1.0 ESP32-S3で完結)**
- スループット: ~20-30Mbps (動画ストリーミング十分)
- 同時接続: 最大10台 (民泊1物件で十分)
- 消費電力追加: +50mW程度 (AP常時ブロードキャスト分)

**v1.1オプション — W5500 有線WAN追加 (+$1.50)**
- SPI接続 Ethernet PHY: 100Mbps有線WAN対応
- 有線ルーターとしても動作 (光回線等に直接接続)
- 基板スペース: USB-C横にRJ45追加 → 基板を65x55mmに拡張

## 3. 通信アーキテクチャ

```
[KAGI Sensor Hub]
    |
    |-- WiFi 2.4GHz --> [KAGI Cloud API] --> [KAGI App]
    |
    |-- BLE 5.0 <--> [SwitchBot Bot/Curtain/Meter]
    |            <--> [Sesame 5 / Sesame Touch]
    |            <--> [Philips Hue Bridge BLE]
```

- **データ送信**: MQTT over TLS (port 8883) or HTTPS POST
- **送信間隔**: センサーデータは60秒ごと、アラートは即座にプッシュ
- **BLEスキャン**: 10秒間隔でアドバタイズパケットをスキャン
- **OTA更新**: HTTPS経由でファームウェア更新

## 4. 基板設計

### サイズ目標
- **基板**: 55mm x 55mm (4層基板)
- **筐体**: 60mm x 60mm x 25mm

### レイヤー構成
| Layer | 用途 |
|-------|------|
| Top | ESP32-S3、センサー IC、LED、ボタン |
| Inner 1 | GNDプレーン |
| Inner 2 | 電源プレーン (3.3V) |
| Bottom | USB-Cコネクタ、CR2032ホルダー、パスコン |

### 配置ガイドライン
- PIRセンサーは基板端に配置（筐体のフレネルレンズ窓に対向）
- SGP40は基板端の通気スリット付近に配置
- INMP441は筐体上面の音孔に対向
- アンテナ（ESP32-S3内蔵）の周囲にGNDプレーンを避ける（キープアウトエリア）

## 5. 筐体設計

### 100台規模: 3Dプリント（SLS/MJF）
- **素材**: PA12 (ナイロン) - 耐熱・耐久性良好
- **仕上げ**: 白色染色 + サンドブラスト
- **コスト**: 約$8-12/個 (JLCPCB 3Dプリントサービス)

### 1000台以上: 射出成形
- **素材**: ABS or PC
- **金型**: 約$3,000-5,000
- **単価**: 約$1-2/個

### 筐体の特徴
- 壁掛け用フック穴 + 両面テープスペース
- PIR用フレネルレンズ窓
- 通気スリット（空気質センサー用）
- 上面に音孔（マイク用）
- USB-C端子は底面
- ステータスLED用の導光材（半透明部分）
- ドア開閉センサーは外部ケーブル（JST PH 2ピン）

## 6. 電源設計

```
USB-C (5V) → ESD保護 (USBLC6-2SC6) → LDO (ME6206A-3.3) → FB1(フェライト) → 3.3V Rail
                                                                                    |
                          LD2410B ← 5V直接給電 (70mA) ← VBUS                       |
                                                                                    |
CR2032 ──→ ダイオードOR (1N5819W) ──→ 470μF バッファ (C5) ──────────────────────>|
             (停電検知: USB電圧モニタリング用ADC GPIO1)
```

- **通常時消費電力**: 約250mW (LD2410B 5V:350mW含む総消費)
- **ピーク時**: 約600mW (WiFi送信 + mmWave同時動作)
- **CR2032バックアップ**: BLEアドバタイズで緊急通知 (15mA、C5バッファで安定動作)

- **通常時消費電力**: 約200mW (60mA @ 3.3V)
- **ピーク時**: 約500mW (WiFi送信時)
- **CR2032バックアップ**: 停電時に1回の通知送信のみ（約30秒動作）

## 7. 安全システム設計 — オープンソース検証可能アーキテクチャ

### 設計原則
- **ハードウェアが責任を取るのではなく、人間の判断を支援する**
- **アルゴリズムは全てオープンソース (GitHub公開) で誰でも検証可能**
- **全イベントはATECC608A署名付きで改ざん不能な監査ログに記録**
- **誤報でも "自動的に緊急連絡先へ通報" は絶対にしない**

### 安全判定ラダー (Safety Ladder)

```
                         ┌──────────────────────────────────────┐
                         │  KAGI Hub センサー検知                │
                         │  mmWave: N分間動き/呼吸なし            │
                         │  + INMP441: 無音                     │
                         │  + ドアセンサー: 閉                   │
                         └──────────────┬───────────────────────┘
                                        │
                         ┌──────────────▼───────────────────────┐
                         │  Tier 1: オーナーへ「確認」通知       │
                         │  「XXさん、異常の可能性があります。    │
                         │   [問題なし] or [確認をお願いします]」│
                         │  タイムアウト: 30分                   │
                         └──────────────┬───────────────────────┘
                                        │ 無応答
                         ┌──────────────▼───────────────────────┐
                         │  Tier 2: 緊急連絡先へ通知             │
                         │  (事前登録された家族・管理会社)       │
                         │  タイムアウト: 30分                   │
                         └──────────────┬───────────────────────┘
                                        │ 無応答 + 事前同意あり
                         ┌──────────────▼───────────────────────┐
                         │  Tier 3: 外部緊急連絡 (119等)        │
                         │  ⚠️ 事前に明示的オプトイン必須        │
                         │  デフォルト: 無効                    │
                         └──────────────────────────────────────┘
```

### オープンソース検証可能な仕組み

```yaml
# safety_config.yml — GitHubで公開・コミュニティレビュー可能
# https://github.com/kacha-io/kagi-safety-protocol

safety_ladder:
  # Tier 1 — オーナー通知 (自動)
  tier1:
    triggers:
      - sensor: mmwave
        condition: no_motion_minutes >= 360  # 6時間
      - sensor: audio
        condition: no_sound_minutes >= 240   # 4時間
      - sensor: door
        condition: closed_since_minutes >= 360
    require_all: true  # AND条件 (誤報防止のため複数センサー一致必須)
    action: notify_owner
    timeout_minutes: 30
    message_template: "open_source"  # テンプレートもGitHubで公開

  # Tier 2 — 緊急連絡先通知 (自動)
  tier2:
    triggers:
      - tier1_no_response: true
    action: notify_emergency_contact
    timeout_minutes: 30
    requires_opt_in: false  # Tier2はデフォルト有効

  # Tier 3 — 外部機関通報 (完全手動オプトイン)
  tier3:
    triggers:
      - tier2_no_response: true
      - owner_explicit_consent: true  # アプリで明示的に有効化が必要
    action: external_emergency  # 119 / 警察 / 管理会社API
    requires_opt_in: true       # デフォルト: 無効
    requires_annual_renewal: true  # 年1回の再同意
```

## 8. ファームウェア概要

- **言語**: Rust (esp-idf-hal, std環境)
- **オープンソース**: https://github.com/kacha-io/kagi-firmware (MIT License予定)
- **RTOS**: FreeRTOS (ESP-IDF内蔵)
- **主要タスク**:
  1. センサー読み取りタスク (1Hz): SHT40/VEML7700/SGP41/BMP280/LIS2DH12/SCD41
  2. mmWaveレーダータスク: OUT割り込み (即時) + UART詳細データ (10秒ごと)
  3. 安全監視タスク: Safety Ladder (section 7参照) — センサー融合で誤報最小化
  4. TinyMLタスク: INMP441音響イベント + LIS2DH12転倒パターン (100KB以下のモデル)
  5. BLEスキャンタスク (0.1Hz): SwitchBot/Sesame/Hue検出
  6. WiFi AP+STAタスク: ゲスト用SSID自動管理 + KAGI Cloud MQTT over mTLS
  7. NFCタスク: ST25DV GPO割り込み → ゲストWiFi資格情報を動的更新
  8. 監査ログタスク: 全イベントをATECC608Aで署名 → クラウドへ送信
  9. LED制御タスク: 家スコアをアンビエントカラーで表現
  10. OTA更新タスク: ECDSA署名検証 (ATECC608A) → Tier-by-tier rollout
- **セキュリティ**: ATECC608A (ECC P-256)
  - MQTT: クライアント証明書認証 (mTLS) → 秘密鍵はATECC608A内で物理保護
  - OTA: ファームウェア署名検証でなりすまし更新を防止
  - 監査ログ: 全安全イベントをハードウェア署名で改ざん不能化
  - デバイスID: 製造時プロビジョニング、クローン防止
- **プロビジョニング**: SoftAP + Webサーバーで初期WiFi設定 + ATECC608A初期化
- **データフォーマット**: MessagePack (JSON比で30%サイズ削減)
- **安全アルゴリズム**: `safety_config.yml` としてGitHubで公開・検証可能

## 9. KAGI Lite -- 詳細設計

### 9.1 設計思想

KAGI Liteは「孤独死ゼロ」のミッションに最も直接的に貢献する製品。
Hub/Proが民泊・施設向けの「高機能モデル」であるのに対し、
Liteは**「独居者の命綱」**として設計する。

```
設計優先順位:
  1. 確実に動くこと (信頼性 > 機能数)
  2. 設定不要であること (高齢者が使えなければ意味がない)
  3. 安いこと (補助金で無償配布できる価格帯)
  4. 小さいこと (部屋に置いても邪魔にならない)
```

### 9.2 BOM構成と原価分析

### 9.5 回路設計要点 (Hub との差分)

Liteの回路はHubのサブセット。以下を削除:

| Hub部品 | Lite | 理由 |
|---------|------|------|
| VEML7700 (照度) | 削除 | 安否確認に不要 |
| SGP41 (VOC/NOx) | 削除 | 安否確認に不要 |
| BMP280 (気圧) | 削除 | 安否確認に不要 |
| INMP441 (マイク) | 削除 | プライバシー懸念を排除 |
| LIS2DH12 (加速度) | 削除 | 転倒検知はmmWaveで代替 |
| SCD41 (CO2) | 削除 | Proモデル専用 |
| ST25DV (NFC) | 削除 | 高齢者にNFCは不要 |
| J3 (水漏れ) | 削除 | 民泊機能 |
| J4 (拡張) | 削除 | Hubのみ |
| D1b (2nd LED) | 削除 | 1個で十分 |

**追加/変更**:
| 変更 | 詳細 |
|------|------|
| SW2を大型化 | 3x4mm → 12x12mm。高齢者が確実に押せるサイズ |
| BZ1ブザー追加 | Tier1通知の聴覚フィードバック + I'm OK確認音 |

**基板サイズ**: 40x40mm (2層基板で十分) → 筐体 45x45x20mm

I2Cバス上のデバイスは4個のみ (SHT40, ATECC608A + 予備2アドレス)。
Hub用ファームウェアと同じI2Cドライバを使用し、不在デバイスはスキャン時に無視。

### 9.6 「I'm OK」ボタンのUX設計

詳細は [SAFETY_ALGORITHM.md](SAFETY_ALGORITHM.md) の「I'm OK ボタン」セクション参照。

#### 物理デザイン

```
            ┌──────────────────────┐
            │                      │
            │     ┌────────┐       │
            │     │ LED(D1)│       │
            │     │ (呼吸) │       │
            │     └────────┘       │
            │                      │
            │   ┌──────────────┐   │
            │   │              │   │
            │   │  I'm OK      │   │ ← 12mm径 大型タクトスイッチ
            │   │  ボタン       │   │    緑色シリコンキャップ
            │   │              │   │    1.5N以下の押圧
            │   └──────────────┘   │
            │                      │
            │  ● BZ1(ブザー)       │
            │                      │
            └──────────────────────┘
                 USB-C (底面)
```

#### ボタンを押した時のフロー

```
ボタン押下
  │
  ├→ BZ1: ピッ (200ms, 確認音)
  ├→ LED: 緑フラッシュ (500ms)
  ├→ WiFi送信: {device_id, "ok", timestamp, signature}
  ├→ KAGI Cloud: ACSを1.0にリセット
  └→ 家族アプリ: 「お父さんがOKを押しました 8:32」
      (通知ではなく、アプリ内のタイムライン更新のみ)
      (毎日通知が来ると家族も疲れる → 「見に行けば分かる」設計)
```

#### 「押さなかった日」のフロー

```
起床推定時刻 + 30分 (ベイズ学習で個人化)
  │
  ├→ BZ1: ピッ...ピッ (2回、リマインド)
  ├→ LED: 緑ゆっくり点滅
  │
  ├─ (さらに60分経過、ボタン未押)
  │   ├→ mmWaveで呼吸検知中? → YES → ACSは0.4程度。まだ様子見
  │   │                       → NO  → ACS < 0.3。Tier1発動
  │   └→ ドアセンサーで外出検知? → YES → 外出中。問題なし
  │
  └─ (24時間経過、ボタン未押、他センサーも反応薄い)
      └→ Tier2: 家族に通知
```
