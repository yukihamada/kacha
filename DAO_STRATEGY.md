# Enabler DAO — ユーザー所有への道

## ビジョン

「使う人が作り、使う人が所有し、使う人が利益を得る」

freeeやMoney Forwardは株主のもの。ユーザーは毎月お金を払うだけ。
Enablerは違う。使う人がオーナーになる。

---

## なぜDAOか

### 今の問題
1. SaaSは使えば使うほどデータがロックインされる
2. ユーザーが増えても、ユーザーには還元されない
3. 会社が潰れたらデータもサービスも消える
4. 開発の方向性にユーザーは口を出せない

### DAOで解決
1. データはローカル → ロックインなし
2. 収益はユーザーに還元
3. コードはオープンソース → 誰でもフォークできる
4. ガバナンスで開発の方向を決められる

## 法規制の壁と回避策

### 日本の規制マップ

| 規制 | 内容 | リスク |
|---|---|---|
| **金商法（金融商品取引法）** | 収益配分を約束するトークン = 有価証券 | トークン保有者に利益を分配すると証券に該当する可能性 |
| **資金決済法** | 暗号資産の発行・交換 = 暗号資産交換業の登録が必要 | 1号暗号資産(BTC等)と交換可能なトークンは登録必要 |
| **景品表示法** | ポイント還元の過大表示 | ポイントの価値を誇大に宣伝するとアウト |
| **出資法** | 元本保証の約束 | 「必ず儲かる」は違法 |

### 何がOKで何がNGか

```
OK:
✅ ガバナンストークン（投票権のみ、利益分配なし）
✅ ユーティリティトークン（アプリ内機能のアンロック）
✅ ポイントプログラム（1ポイント=1円の割引）
✅ オープンソースへの貢献報酬（バウンティ）
✅ 協同組合（法律に基づく利益分配）
✅ NFT（コレクティブル、ガバナンス投票権）

NG:
❌ 「トークンを持つと利益が出ます」と宣伝
❌ トークンの価格上昇を示唆するマーケティング
❌ 利益分配を約束するトークン販売
❌ 未登録の暗号資産交換
```

## 3フェーズ戦略

### フェーズ1: コントリビューションエコノミー（今すぐ）
**法的リスク: ゼロ**

ユーザーの貢献を記録し、見える化する。金銭的価値は付けない。

#### 仕組み
```
コントリビューション = ポイント（ENAIポイント）

獲得方法:
- アプリを使い続ける → 1日1ポイント
- バグ報告 → 10ポイント
- GitHub PR がマージ → 100ポイント
- レビュー投稿 → 5ポイント
- 翻訳貢献 → 50ポイント
- 友達紹介 → 20ポイント
- ハウスマニュアルのテンプレート作成 → 30ポイント
```

#### ポイントの使い道（Phase 1）
- Pro機能の無料期間
- アプリ内バッジ/ランク表示
- ガバナンス投票権（機能リクエストの優先順位）
- Enablerグッズ（ステッカー、Tシャツ）

#### 実装
- アプリ内に「マイコントリビューション」画面
- GitHubのコミット/Issue/PRと連携
- リーダーボード（匿名可）

**これは完全に合法。ポイントに金銭的価値がないから。**

### フェーズ2: 協同組合モデル（6ヶ月後）
**法的リスク: 低（既存の法制度内）**

日本の「事業協同組合」or「企業組合」として法人化し、
ユーザーを組合員にする。

#### 仕組み
```
Enabler協同組合

組合員 = Pro/Businessプランの利用者
出資金 = ¥10,000（一口）
議決権 = 一人一票（出資額に関係なく平等）
配当 = 事業の利益に応じて組合員に配分（年1回）
```

#### なぜ協同組合か
1. **完全合法**: 中小企業等協同組合法に基づく
2. **利益分配OK**: 組合員への配当は法律で認められている
3. **一人一票**: DAOの精神と一致
4. **税制優遇**: 法人税が低い（年800万円以下は15%）
5. **日本で実績豊富**: 農協、生協、信用組合は全部これ

#### 具体的な利益分配
```
Enabler協同組合の年間収益: 2,000万円（仮定）
- 開発費: 800万円
- サーバー費: 100万円
- マーケティング: 300万円
- 組合員配当: 500万円（組合員1,000人なら一人5,000円/年）
- 内部留保: 300万円
```

**「SaaSに月980円払って何も返ってこない」vs「Enablerに月980円払って年末に5,000円返ってくる」**

これだけで差別化になる。

### フェーズ3: トークンガバナンス（1年後〜）
**法的リスク: 中（慎重な設計が必要）**

ENAIトークンをガバナンストークンとして位置付ける。
利益分配はフェーズ2の協同組合で行い、トークンは投票権のみ。

#### トークンの役割（利益分配しない）
```
ENAIトークン = ガバナンス投票権

投票できること:
- 次に開発する機能の優先順位
- マーケティング予算の配分
- 新規アプリの開発承認
- パートナーシップの承認
- 利益分配の比率（協同組合として）

投票できないこと:
- トークンの買い戻し（自社株買い的なもの）
- トークン価格に影響する施策
```

#### なぜこれが合法か
- トークン自体に利益分配機能がない → 有価証券に該当しない
- ガバナンス投票はプロジェクトの方向性決定 → ユーティリティ
- 利益分配は協同組合の法的枠組み内で実施
- トークンとお金の交換は行わない（獲得はコントリビューションのみ）

## 「匂わせ」のやり方

直接的に「儲かります」は言えない。でも方向性は示せる。

### OKな表現
```
✅ 「Enablerはユーザーが所有するソフトウェアを目指しています」
✅ 「オープンソースで、コミュニティが開発の方向性を決めます」
✅ 「コントリビューションを記録し、将来的なガバナンスに活用します」
✅ 「使えば使うほど、あなたの声が大きくなります」
✅ 「私たちのゴールは、ユーザーとチームの境界をなくすことです」
✅ 「協同組合モデルでの運営を検討しています」
```

### NGな表現
```
❌ 「トークンの価値が上がります」
❌ 「投資として利益が出ます」
❌ 「早く参加するほど得です」
❌ 「トークンを買ってください」
```

### サイトへの反映案

pasha.run のフッターに:
```
Enabler DAO — ユーザーが所有するソフトウェアへ
```

コードはGitHubで公開されており、誰でも検証・貢献できます。
将来的にはユーザーガバナンスによる運営を目指しています。
```

README.mdに:

```

## 先行事例

| プロジェクト | モデル | 成功度 | 学び |
|---|---|---|---|
| **Uniswap** | ガバナンストークン | ◎ | トークン配布でユーザー爆増。ただし投機的 |
| **Gitcoin** | 公共財ファンディング | ○ | OSSへの資金提供。持続可能性に課題 |
| **Mirror.xyz** | ライタートークン | △ | メディア×トークン。規制で縮小 |
| **Stocksy** | フォトグラファー協同組合 | ◎ | iStockの代替。利益を写真家に還元 |
| **REI** | 消費者協同組合 | ◎ | 年間配当あり。会員2,300万人 |
| **Mondragon** | 労働者協同組合 | ◎ | スペイン最大の協同組合。売上1.2兆円 |
| **Home Assistant** | OSS + クラウド課金 | ◎ | Nabu Casaが有料。OSSは無料 |

**最も参考になるのは REI + Home Assistant のハイブリッド:**
- Home Assistantのように、OSSは無料で誰でも使える
- REIのように、有料メンバーは組合員として配当を受ける
- ガバナンスはGitHub + トークン投票のハイブリッド

## 一言でまとめると

**「freeeに月1,000円払って株主だけが儲かる世界」から、
「Enablerに月1,000円払って自分も配当をもらえる世界」へ。**

これがEnablerの存在意義。
ローカルファースト × オープンソース × ユーザー所有。

法的にはまず協同組合でやる。トークンはガバナンスのみ。
「匂わせ」はビジョンとして語る。実装は法の範囲内で着実に。
